Паттерн Builder (Строитель) является порождающим паттерном проектирования, который позволяет создавать сложные объекты шаг за шагом.
Он позволяет отделить процесс конструирования объекта от его представления, чтобы один и тот же процесс конструирования мог создавать разные представления объекта.

Принцип работы паттерна Builder следующий:

1. WebsiteBuilder - Определение интерфейса строителя: Это интерфейс, который определяет методы для пошагового создания объекта.
2. VisitCardWebsiteBuilder, EnterpriseWebsiteBuilder - Создание конкретного строителя: Класс конкретного строителя реализует интерфейс
    строителя и предоставляет методы для каждого шага конструирования объекта.
3. Director - Создание класса директора (необязательно): Директор класса может управлять процессом конструирования, вызывая методы строителя в правильном порядке.
4. Website - Создание класса продукта: Класс продукта представляет объект, который должен быть построен. Он содержит свойства и методы, которые определяют его состояние и поведение.
5. MainTest - Клиентский код: Клиентский код создает объект строителя и вызывает его методы для пошагового конструирования объекта. Затем клиент может получить готовый объект от строителя.

Паттерн Builder позволяет создавать объекты с сложной структурой, упрощает добавление новых шагов конструирования и предоставляет контроль над процессом создания объекта.
Он особенно полезен, когда объекты имеют множество настраиваемых параметров или требуют сложной логики конструирования.
