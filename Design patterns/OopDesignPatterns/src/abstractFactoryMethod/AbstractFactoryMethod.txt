
Шаблон: Абстрактная фабрика (Abstract Factory)
=======================================================================================================
Цель:

Создание интерфейса, для создания множества взаимосвязанных или взаимозависимых объектов, без жёсткой
привязке к конкретным классам.

=======================================================================================================

Для чего используется:

Для создания множеств взаимосвязанных объектов.

=======================================================================================================
Пример использования:

- система не должна зависеть от метода создания, компоновки и представления входящих в неё объектов;
- входящие взаимосвязанные объекты должны использоваться вместе;
- система должна конфигурироваться одним из множеств объектов, из которых она состоит;
- нам необходимо предоставить множество объектов, раскрывая только их интерфейсы но не реализацию.

=======================================================================================================

Паттерн Abstract Factory (Абстрактная фабрика) расширяет идею паттерна Factory, позволяя создавать семейства связанных объектов, а не только одиночные объекты.
Он предоставляет интерфейс для создания различных типов объектов, связанных между собой, без привязки к конкретным классам.

Принцип работы паттерна Abstract Factory следующий:

1. Определение абстрактного базового класса (или интерфейса), который определяет методы создания объектов различных типов. // Developer
2. Создание конкретных классов, которые наследуются от базового класса и предоставляют реализацию методов создания объектов. // JavaDeveloper, PhpDeveloper
3. Создание абстрактного фабричного класса (или интерфейса), который определяет методы для создания семейств связанных объектов. // ProjectTeamFactory
4. Создание конкретных фабричных классов, которые реализуют абстрактный фабричный интерфейс и предоставляют реализацию методов создания объектов семейства. // BankingTeamFactory, WebsiteTeamFactory
5. Клиентский код работает с абстрактными фабриками и вызывает их методы для получения нужных объектов семейства. // MainTest
